# Honeypot Deployment and Threat Analysis Using Cowrie and Wazuh SIEM

## Overview

This project demonstrates the deployment of a secure SSH honeypot using **Cowrie**, integrated with **Wazuh SIEM** for centralized log collection, threat detection, and attacker behavior analysis within an isolated virtual lab environment.

A controlled attack was simulated using **Kali Linux** to observe common attacker techniques, including brute-force login attempts, command execution, and file download activity. Security events generated by the honeypot were collected by Wazuh, analyzed in real time, and enriched using AI-assisted analysis mapped to the **MITRE ATT&CK Framework**.

The project highlights how honeypot technology, SIEM platforms, and threat intelligence can be combined to improve security monitoring, attacker analysis, and cyber threat detection.

---

## Project Objectives

- Design and deploy an isolated cybersecurity lab environment.
- Deploy a Cowrie SSH honeypot using Docker.
- Simulate SSH attacks from Kali Linux.
- Collect and centralize honeypot logs using Wazuh SIEM.
- Develop custom detection rules for suspicious attacker activity.
- Analyze attacker behavior through log investigation.
- Map observed techniques to the MITRE ATT&CK Framework.
- Demonstrate AI-assisted threat analysis with manual validation.

## Attack Simulation

A controlled attack was performed from a Kali Linux system against the Cowrie SSH honeypot.

The simulated attacker performed activities including:

- SSH brute-force login attempts
- Successful authentication using weak credentials
- System reconnaissance commands
- Command execution
- File download attempts
- Post-compromise interaction

All attacker activities were safely contained within an isolated laboratory environment.

---

## Threat Detection Workflow

1. Kali Linux initiates SSH attacks against the Cowrie honeypot.
2. Cowrie records authentication attempts and attacker commands.
3. Wazuh Agent forwards honeypot logs to the SIEM server.
4. Wazuh analyzes incoming events using custom detection rules.
5. Security alerts are generated for suspicious activities.
6. Logs are visualized through the Wazuh Dashboard.
7. AI-assisted analysis enriches attacker behavior and maps techniques to MITRE ATT&CK.
8. Results are manually validated to ensure analytical accuracy.
